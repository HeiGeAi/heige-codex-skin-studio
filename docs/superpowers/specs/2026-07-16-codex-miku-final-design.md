# Codex Miku Theme v5 最终设计规范

## 目标与真相源

最终主题以用户提供的高清参考图 `assets/miku-reference.png` 为唯一视觉真相源，在真实 macOS Codex Desktop 中复现高饱和青蓝、粉紫、雾白玻璃与初音未来主视觉。验收必须基于真实 `/Applications/ChatGPT.app` 运行截图，HTML 预览只能用于开发检查。

参考截图按 16:9 画布衡量。左栏约占视口 20%，主区约占 80%；主区从顶部品牌栏连续延伸到输入框，不允许出现大面积纯白断层。主题强调色为青色 `#19c9e5`、蓝色 `#0e438d`、粉色 `#ed6ec1`、紫色 `#ad7ed5`，正文使用深海军蓝，玻璃面板使用高透雾白。

## 视觉层级

主题分为三层，禁止重复叠图：

1. 纯 Hero 层：由高清人物参考图生成，只包含人物与霓虹背景，不烘焙标题、卡片、输入框或拍立得。
2. 真实界面层：Codex 自身的侧栏、建议卡片、消息区、审批卡片、弹窗和 composer 由 CSS 玻璃化，保留真实交互与可读性。
3. 独立拍立得层：右下角只使用 `assets/miku-polaroid.png`，其内容、裁切、颜色和 SHA-256 `d30fb69fcd0db2c02b26d6927125caf2f89a178945199ddd9ea86f9853ecd042` 不得改变。

左栏使用青白到薰衣草渐变、细青描边和星光纹理，宽度目标为视口 19% 至 21%。顶端形成 `Miku Codex` 品牌区；一级菜单、项目分组、任务分组、选中胶囊、偏好设置与用户区保持清晰层级。侧栏装饰图不能烘焙真实按钮或分隔线，以免与动态 DOM 重叠。

主区 Hero 目标高度约为视口 58% 至 61%，人物完整露出上半身。四张真实建议卡在 Hero 下部形成等宽四列玻璃卡，带青色外描边、粉紫内光、大图标、两行深蓝标题与粉色点缀。composer 宽度目标为主区可用宽度的 72% 至 76%，与主画布保持同一玻璃体系。

## 主窗口与宠物窗口隔离

主题只能在包含 `.app-shell-left-panel` 的主窗口启用。`html`、`body`、`#root`、`#root::before`、`#root::after` 的所有背景和装饰都必须受主窗口选择器约束。宠物 WebView、设置预览及其他独立窗口保持透明或产品原生背景，不得出现大人物矩形图、点阵画布、`MIKU 488137` 伪元素或拍立得。

配套宠物使用本地最新 `Miku Future` 原生 v2 sprite，1536×2288、8 列×11 行，当前 SHA-256 为 `3fc484a68dad02dc44fd354dbcac21ae78814d29351eebf4d200e30298da5c62`。安装器将其放入 `~/.codex/pets/miku-future/`，并在配置缺少 `selected-avatar-id` 时追加，存在时精确更新，已为目标值时保持幂等。

## 安装、安全与兼容

主题继续使用固定尺寸 ASAR 替换，不修改资源偏移，不临时重签应用。安装前验证 Codex 版本、构建号、入口、CSS 容量及四个图片槽；首次安装保存完整 ASAR SHA-256 备份，安装与恢复均使用原子交换和写后验证。

排队安装任务必须是真正的一次性任务。它在 Codex 主进程退出后等待 Renderer 与 Service 完全收尾，仅对明确的“Codex 尚未完全退出”错误做有限重试，随后安装、验证、重新打开 Codex并卸载自身任务。任何其他错误立即失败并保留精确日志。

主题 CSS 压缩后不得超过旧官方基线的 8003 字节，以同时支持原始官方 ASAR 与本机已被其他主题工具扩容的同构 ASAR。版本号不是唯一门禁，安装器还必须验证结构、容量、槽位与完整归档状态，拒绝不安全覆盖。

## 最终验收

最终交付同时满足：

1. `npm test`、Shell 语法、Skill 校验、ZIP 完整性与 `git diff --check` 全绿。
2. 真实应用检查显示 `installed: true`、`installedVersion: 5`、四张主题素材均为 `installed: true`。
3. 1920×1080 或等比例 16:9 实机截图核对侧栏宽度、Hero 高度、人物裁切、四卡、composer、ID `488137`、右下拍立得和宠物隔离。
4. 恢复后与原始 ASAR SHA-256 一致，再次安装仍通过，且一次性 launchd label 已消失。
5. Skill 从干净 clone 可重建，最终宠物、主题源码、Skill 包和 Release 附件哈希一致。
6. 推送 GitHub `main`，创建 `v5.0.0` Release，并从 GitHub 下载附件复验。
